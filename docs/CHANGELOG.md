# 만신 — 설계 변경 이력

설계가 **왜** 지금 모습이 됐는지의 논거 기록. 최신이 위. 규칙·상한을 바꾸기 전에 해당 항목의 논거를 먼저 읽을 것.

> **버전 체계가 두 가지다:**
> - **설계 버전 `v0.x`** — 이 문서의 개정 차수. 시스템·규칙 설계가 바뀔 때 올라간다.
> - **빌드 버전 `v10.x`** — `index.html` 개발 반복 차수 (아티팩트 배포 라벨과 커밋에 사용). 버그 수정·UX 폴리시마다 올라간다.
>
> 둘은 독립이다. 예: 설계 v0.6.1 시점의 게임 빌드는 v10.1~v10.2.

## v0.7 (빌드 v10.4 — 난이도 이원화·경로 전략·명시적 UX)

**배경**: 첫 풀 클리어 리포트 "한 번에 4막 클리어, 시시함" + 외부 리뷰 합의 — 적 수치보다 회복·보상·성장 경제가 너무 넉넉해 실수와 경로 선택의 대가가 다음 막까지 남지 않는 것이 원인. 적 HP 일괄 상향 대신 경제 통제 → 데이터 수집 → 수치 조정 순서로 접근.

- **난이도 이원화**: 이야기(기존 수치 보존, 금제 비활성) / **본굿(표준·추천, 기본값)**. 본굿 수치 — 당산목 45%→30%, 막 종료 자동회복 40%→15%, 천도 30%·최대+3·방어8힘1 → 20%·최대+2·방어6, 명다리 45%/30전→30%/35전, 일반전 무구 드래프트 50%→30%, 안전 기연 회복 절반(+엿장수 거절은 무보상). 난이도는 런 저장에 귀속. **적 능력치는 불변** (smoke18이 보증)
- **지도 UX**: 재탭 확정 제거 — 탭은 언제나 미리보기, 확정은 스티키 바텀시트의 「이 길로 간다」로만 (첫 행은 「첫 굿판 시작」). 먼 노드도 열람 가능(확정 불가). **위험 태그**(연타·반사·관통·동티·한(恨)·약화·성장·흡혈·철갑·회복·거울 — 동티와 恨은 별개 태그)를 실제 사용될 패턴에서 유도해 예고 — 경로 전략의 재료
- **막별 전략**: 4막(뒷전거리) 재등장 귀신은 **상위 패턴(patUp)** 사용 — 기제 최대 2개 조합(처녀 약화+저주 / 도깨비 철갑+연타 / 두억시니 성장+연타 / 이무기 흡혈+저주 / 삼목구 연타+관통 / 불가사리 철갑+반사 / 저승사자 관통+저주). 숨은 카운터·손패 읽기 금지, 태그·의도 상세로 전부 사전 고지
- **보스 기제**: 구미호 50%에서 **전용 2페이즈 패턴 교체**(3연타·성장·흡혈·저주, pi 리셋, 힘+2)+의도 즉시 갱신(공정성). 지네각시 50%에서 백족 단계 즉시 +1(1회). 장산범 거울 상세에 대응법 명시
- **명시적 튜토리얼·용어**: 타이틀 재구성(이어하기 최상단·새 굿 2단 덮어쓰기 확인·처음 배우기·도움말·용어 사전). 상황별 팁 8종(ID별 1회, 초기화 가능). **「쉬운 행동 설명 — 무속 용어」 이중 표기** + P0로 신단 원한/전투 恨 분리(HUD 칩 탭 설명, 원한 3 도달 시 임계 예고). 보상 3택에 선택 전 실제 변화량(원한·동티·체력 전후) 표시
- **런 계측(runStats, 로컬 전용)**: 난이도·몸주신·막별 시작/보스 진입 HP·받은 피해·전투 외 회복·평균 전투 턴·경로 분포·봉인/천도/치성·사망 막·승패. 종료 화면 요약+JSON 복사. mapVersion 4
- **표준 난이도 목표치**(가설, 계측으로 검증): 1막 통과 80~90% / 2막 60~75% / 3막 40~55% / 첫 클리어 25~40%, 일반전 3~5턴 / 보스 6~9턴 / 보스 진입 체력 55~80%

## v0.6.3 (빌드 v10.3~10.3.2 — 전투 연출·오디오 개편)

- **타격감 기본기(1세대 문법)**: 히트스톱(60~110ms) + 피격 순백 실루엣 + 화면 플래시/반전 + 절차 임팩트 스프라이트(pixelate 파이프라인 통과 — 디더·외곽선이 캐릭터와 자동 일치) + 데미지 숫자 팝. 논거: 화려함은 파티클 양이 아니라 **예비동작→접촉→히트스톱→회복 리듬**에서 나온다 (외부 리뷰 합의)
- **1세대식 대각 전투 구도**: 적 우상(기준선 y186) ↔ 적 정보 좌상, 무당 좌하 ↔ 무당 정보 우하(이름·HP 한 줄 슬림형), 의도 칩은 적 정보 아래 — 정보창이 캐릭터를 덮던 문제의 구조적 해결. 초상화는 콘텐츠 경계 크롭으로 키 큰 신령 머리/넓은 꼬리 잘림 제거, 그슨대는 가면을 상시 가시 높이로 이동
- **광과민성 배려**: 타이틀에 화면 번쩍임 전체/약하게/끔(`ms_flash`) — `prefers-reduced-motion`은 광과민성 설정이 아니므로 별도 옵션. 다단히트는 마지막 타에만 화면 플래시
- **카드 fx 6계통**: slash / talisman(부적 투사체 — 적중 연출을 FX age 기반 `onContact`로 도착 시점에 발화, 히트스톱과 같은 시간축 + 전투 종료 잔류 콜백 가드) / hanMist(恨 문양) / seal(烙 낙관) / ward(결계 파문) / heal(서천꽃) — 색과 실루엣으로 구분, 공격 fx와 같은 계통의 자동 상태 fx는 중복 재생 가드
- **적 모션 3유형**: wraith(부유·찢어지는 피격) / beast(웅크렸다 도약) / giant(화면이 먼저 울리고 몸이 뒤따름) — `SPRITE_META`로 개체 보정(구미호 본색 꼬리는 렌더 경계 내 1.12배 상한)
- **죽음 디더 디졸브**(4×4 베이어 3단계 + 재 입자) · 상태 상시 시각화(恨 wisp는 수치 비례, 낙인은 烙 문양 맥동)
- **오디오 전면 절차 합성**: 계통별 시전→적중→잔향 3레이어, 연타 first/mid/last 강약(첫 타 강조·중간 축약·마지막 1.2배), 음악 버스 분리 후 타격/굿거리/처치 시 3~5dB·0.3~0.5s 덕킹. 전투 BGM = 저역 드론 + 밴드패스 밤바람 + 성근 계면조 종소리. **CC0 내장 대신 절차 합성을 택한 논거**: 단일 파일·의존성 0 원칙 유지(+0 용량), 라이선스 관리 불요, 톤 일관성
- 검증: smoke16 — FX 프레임 경계·fx 태그 전수·가짜 AudioContext 계측(버스 연결/강약 볼륨/덕킹 램프/연타 순서/투사체 지연)

- **무속 용어 체계**: 런 = 밤길의 **진오기굿** / 막 = **부정거리 → 영실거리 → 뒷전거리** / 에너지 = **신력(神力)** / 상점 = 당골네 할멈의 장터(엽전) / 반복 난이도 = **금제(禁制)** 0~3

- **해원(解冤) 퀘스트**: 신물(족두리·꽃신·제삿밥)을 구해 사연이 맞는 귀신에게 건네면 전투 없이 **해원 봉인(원한 0)** 또는 **해원 천도(대회복)**. 혼수함 이벤트·제삿밥 짓기로 신물 획득, 지도에 '解' 표시
