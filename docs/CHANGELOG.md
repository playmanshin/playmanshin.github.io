# 만신 — 설계 변경 이력

설계가 **왜** 지금 모습이 됐는지의 논거 기록. 최신이 위. 규칙·상한을 바꾸기 전에 해당 항목의 논거를 먼저 읽을 것.

> **버전 체계가 두 가지다:**
> - **설계 버전 `v0.x`** — 이 문서의 개정 차수. 시스템·규칙 설계가 바뀔 때 올라간다.
> - **빌드 버전 `v10.x`** — `index.html` 개발 반복 차수 (아티팩트 배포 라벨과 커밋에 사용). 버그 수정·UX 폴리시마다 올라간다.
>
> 둘은 독립이다. 예: 설계 v0.6.1 시점의 게임 빌드는 v10.1~v10.2.

## 빌드 v10.10 (설계 v0.9 — 전투 판단을 먼저 보이는 UI/UX 폴리시)

**배경**: 모바일 360×640 기준으로 다시 읽었을 때 시작 화면의 플레이·학습·설정이 같은 시각 무게로 쌓였고, 첫 전투 안내는 2.8초 토스트라 행동 전에 사라질 수 있었다. 전투 의도 자체는 정확했지만 플레이어가 실제로 궁금한 **다음 총피해 → 현재 방어 → 체력 손실**은 여러 창을 오가며 계산해야 했다. Slay the Spire의 적 의도/턴 행동 위계와 Balatro의 계산 결과 중심 피드백·카드 촉감을 참고하되, 만신의 대각 굿판·부적·오방색 문법으로 번역했다.

- **시작 화면 점진 공개**: 저장이 없으면 「처음 배우기」, 저장이 있으면 「이어서 야행한다」만 primary. 소리·번쩍임·난이도·금제는 「설정·난이도」 아래 접어 첫 행동을 3초 안에 고를 수 있게 했다. OS별 이모지 편차가 큰 메뉴 아이콘은 燈·初·書·設·音·光·衡 한자 표기로 통일
- **행동 결합형 첫 전투 코치**: 부적 살피기 → 귀신 수 읽기 → 턴 종료 3단계를 2.8초 토스트가 아닌 지속 스트립으로 표시하고, 지금 만질 카드/의도/턴 종료를 직접 강조. 기존 상황별 팁 seen 키와 호환하며 도움말 초기화도 유지
- **전투 판단 스트립**: 빈 공간(신단 바와 손패 사이)에 다음 총피해·타수·관통·현재 방어·예상 체력 손실을 상시 표시. 전투 실행/의도 칩과 동일한 `calcIncomingHit()`을 경유하고, 그믐밤 수치 은폐·씨름 스킵도 그대로 반영한다. 규칙·밸런스·저장 스키마 변경 없음
- **카드 스캔성**: 소유 신령 색 위에 殺(공격)·守(방어)·生(회복)·陣/術(지속/술법)·祟(동티) 문양을 중첩해 색각과 작은 글씨에만 의존하지 않는다. 카드/의도에 Enter·Space 조작과 구체적 접근성 라벨 추가
- **전투 피드백·캐릭터 실루엣**: 카드를 내는 순간 이름/역할 시전 배너를 짧게 남기고, 다단히트 피해 숫자를 계단 배치하며 큰 피해 숫자의 크기·외곽선을 강화. 무당은 색동 허리띠·지전 꼬리·비녀를 fillRect 블록 매스로 추가해 1.1배 전투 스케일에서도 적과 구분되는 가로/세로 앵커를 얻었다. 번쩍임·reduced-motion 정책은 유지
- **검증**: smoke26 신설 — 시작 메뉴 행동 위계, 판단 스트립의 다단/방어/관통/그믐/스킵 파리티, 카드 역할 5종, 코치 교대, 시전 배너와 키보드 배선 assert. smoke20의 팁 배선 검증은 지속형 코치 seen 기록을 포함하도록 확장

## v0.9 (빌드 v10.9 — 빌드 다양성 P1: 덱이 나를 고르는 경제)

**배경**: v0.8이 축의 수확처(payoff)를 만들었다면, v0.9는 그 축을 **만날 확률**을 설계한다. 기존 무구 보상은 단일·균등·무희귀도 복권이라 빌드 결정이 전부 신령 축에만 실려 있었다(분석: "카드 보상 축은 어떤 빌드에도 같은 복권"). StS의 희귀도 게이팅+pity, Balatro의 참조 밀도를 이식하되, 순수 랜덤 슬롯을 남겨 발견의 재미(sought)를 보존한다.

- **무구 희귀도 3층**: 커먼 14(무색 주식) / 언커먼 14(축 enabler) / 레어 9(build-around payoff·신령부 스케일러 — 프로스트 오브의 교훈 "payoff 빈곤이 축을 죽인다" 역적용). 드래프트 카드에 ★(레어)·◆(언커먼) 표기
- **드래프트 3슬롯 규칙**: 슬롯A=커먼 균등 / 슬롯B=희귀도 롤(커먼60·언커먼40)+**시너지 가중**(w=1+2×출전 신령 축과의 겹침) / 슬롯C=커먼55·언커먼30·레어15%+pity. **정예 전투는 슬롯C 레어 확정** — StS "정예=유물"의 만신 번역, 레어 주 공급 채널. 중복 감쇠(2장 보유 제외·1장 ×0.5), 보장(셋 다 무관 축이면 슬롯B를 겹침≥1 후보에서 재추첨 — 같은 희귀도→비레어→전체 순으로 pity 회계 보존)
- **pity 「달이 찬다」**: 레어 미제시 드래프트마다 +15%p, 제시 시 15%로 리셋, 런 저장 귀속. 30%p 이상이면 드래프트 부제에 "달이 만월에 가깝다 — 깊이 잠든 무구가 몸을 뒤척인다" — 숨은 카운터 금지 원칙의 보상 측 연장(상태를 서사로 고지)
- **축 태그 체계**: 무구 전수에 `CARD_AX`(11축: 恨·힘·낙인·방어·회복·패·동티·제물·보존·신명·살), 신령 전수에 `SPIRIT_AX`(첫 원소가 주축). 드래프트·장터 카드에 축 라벨 표기, 내 신단과 겹치면 금색 강조 — "덱이 나를 골랐다"의 시각화. **편성이 카드 경제의 조준 장치**가 된다(편성을 바꾸면 다음 드래프트의 슬롯B가 따라온다)
- **장터 연동**: 재고 부적 2장 중 1장에 동일 시너지 가중, 미리보기를 cardVals 경유로 통일
- **runStats v3**: 드래프트 제시·픽·스킵 기록(`drafts`) + 최종 덱 목록(`deckList`) — 픽률·승리덱 등장률 2지표(StS GDC 방법론) 수집 기반. 목표: 전 카드 픽률 ≥10%(미달은 삭제가 아니라 시너지 간선 추가부터), 축 점유율 60% 초과 시 cannibalize 신호
- **무구 강화 접점 개통**: v0.8 isUp 개통의 후속 — 장터 「주사(朱砂) 덧칠」(40전, 방문당 1회, 무소속 부적 1장 강화 — 신규 부적 35전보다 비싼 프리미엄: 강화는 덱 희석 없는 파워) / **치성 확장** 「무구 일습에 치성」(무소속 미강화 ≥2장일 때, 2장 선택 강화 — 신령 치성 3장 일괄·무선택과의 등가는 "장수는 적지만 선택권"). 봉인/천도/치성 삼자택일에 범용 무구 덱도 고객이 된다. 덧칠 목록은 강화 전후 수치를 함께 표시(보상 미리보기 원칙)
- **귀신 킷 리워크 — 바닐라에 오라의 문장을**: 카드 전수 분석의 처방("귀신 킷별로 오라 키워드와 연결하는 리워크가 가장 효율적") 실행. 조건부 보너스 `v.cond` 단일 평가기(11종 조건, 전부 공개 정보 — 숨은 카운터 금지) 도입, 킷 11장 등가 재배분 리워크(기본 −1~−2, 조건 +3~+6): 들이받기(체력 절반↓ +6)·쇠비늘(잃은 체력 비례)·낄낄(턴 카드 수)·밤도망(신력 잔존)·방망이질(적 방어 벌주기)·어둠뭉치(지닌 방어)·물장막(적 약화)·이승 미련(적 의도 공격)·저승 이빨(첫 공격)·명부 호명(적 恨)·말뚝 치기(방어 5+). 무축 바닐라 41%→약 30%
- **무색 오라 3건 → 축 payoff 전환** (v0.7.1 "누적 대신 전환"의 귀신 레이어 이식): 불가사리 불가살→**쇠먹기(食鐵)**(턴이 넘어갈 때 버려지는 방어 3당 적 피해 1, 최대 5 — 이월분 제외로 이중이득 차단), 묘귀 아홉목숨→**보은(報恩)**(회복한 턴 첫 공격 +3, 산신 몸주 +5 — 죽음 안전망은 초혼·명줄이 대체), 몽달 곡소리→**서러운 곡**(약화 적에게 공격 카드당 +3 — 약화 14장의 첫 수확처). 수비 오라 2종이 사라지는 대신 방어·회복·살 축이 각각 공격 출구를 얻는다
- **축 payoff 2차 3장**: 명줄 당기기(전투 누적 회복 3당 +1, 최대 +10 — 탯줄의 소비형 카운터와 분리된 `battleHealTotal` 총계 참조로 상호 간섭 차단), 낙인화로(신령부: 매 턴 첫 공격이 낙인 각인 — enabler 병목 해소, 레어), 천기누설(덱탑 공개 → 공격이면 공수 +6, 아니면 드로우 — 꽝 없음, 점괘와 2카드 콤보)
- **듀엣 15건 — 신령이 서로를 지명한다** (기존: 크로스 참조 0건, "합이 맞는" 조합 8~12%): 카드 듀엣 8건은 `cardVals` 단일 경유(표시=실제 파리티 자동, 대기 중에도 「합(대기)」로 카드에 상시 표기 — 숨은 카운터 금지), 굿거리 듀엣 4건은 `gutFxOf()` 헬퍼(사혼굿 처녀×몽달 전투화 · 바리×저승사자 비례 25% · 삼신×객귀 회복 16 · 칠성×달걀 드로 4), 오라 확장 2건(몽달 빙의×물귀신 = 턴 시작 약화 +1 · 산신 몸주×보은 +5), 배치 특수 1건(염매×허깨비 개전 동티 — **적대 검토로 2→1장**). 서사는 전부 민속 모티프(사혼·저승길 동무·제삿밥·장물아비…). **밸런스 수정 반영**: 광란 듀엣은 "타수 +1"이 아니라 **첫 타 +4**(타수 증가 금지 원칙 — 다단×무상한 힘 자가발전 차단), 명부 호명 듀엣 +5→+4
- **합(合) — 편성 자체가 개전 보너스**: 신령마다 주축(SPIRIT_AX 첫 태그). 출전 귀신 2위 주축 일치 = **대합**(恨+4/방어+6/회복5/드로+1/힘+1/약화2), 몸주신 주축 일치 = **소합**(절반, 힘 축은 공수+3, 1회) — 중첩 가능, 편성 화면에서 사전 예고. (몸주, 귀신2) 918 편성 중 명시 시너지 발동 비율 8~12% → 약 45~55%
- **개전 恨 공유 예산 6 (적대 검토 치명 문제 처방)**: 길잡이·점지·대합·소합 등 모든 개전 恨 소스가 예산 6을 공유 — 바리 hanNoDecay와 결합한 "카드 0장 자동승"(개전 恨 12 적층) 원천 차단. smoke25가 길잡이+대합+소합=8 요구 시나리오에서 6 클램프를 assert
- **구현 후 적대 검증 반영 (2차)**: ① 개전 예산이 healPlayer 훅(꽃값·삼신끈)으로 우회되던 구멍(실측 9) — 예산을 `gainHan` 자체에 내장(개전 구간 한정 활성)해 "모든 개전 소스 공유"를 코드로 보증 ② rareLuck이 새 굿에서 리셋되지 않던 런 간 pity 누수 — initRunStats에서 리셋(계측 오염 방지) ③ 천기누설 공격 판정에 dmgPerSpirit 누락(신장대 오분류) ④ 씨름(skipTurn) 시 의도 칩 '쿵' 표기(이승 미련 condBonus와 파리티) + 거울 수(m)는 공격으로 판정 통일 ⑤ 무구 치성 계측 기록을 첫 획 확정 시점으로(취소 시 유령 기록 방지) ⑥ **장터 희귀도별 가격**(커먼 35 / 언커먼 45 / 레어 60전) — 균등 35전이 드래프트 레어 게이트(15%+pity·정예 확정)를 언더컷하던 문제
- **계측 주시 항목 (30런 게이트에서 판정, 선(先)너프 금지)**: 방어참조 적층(어둑시니 이월 12 위에 지신밟기+어둠뭉치+말뚝치기+승천 = 턴 ~47, 단 4조각 조립 요구 — 터지면 방어참조 가산에 턴당 합산 캡 12 후보) / 명부 호명 3중 적층(cond+듀엣+세번째눈 = 27/2신력) / 낙인화로(u)×인주(2턴 낙인 8 만재) / 개전 회복이 보은을 1턴 상시화(문언 부합 — 의도로 확정) / pity가 픽이 아닌 제시 기준 리셋(정예 확정 포함 — 의도로 확정)

## v0.8 (빌드 v10.8 — 빌드 다양성 P0: 죽은 축에 수확처를)

**배경**: 외부 비교 조사(StS·Balatro·Monster Train) + 카드 107장 전수 분석 — 무축 바닐라 41%, 명시 참조 카드 7.5%(8장), "쌓기만 하고 수확할 곳 없는" 축이 5개(방어 25장/수확 0 · 자해 7/0 · 소멸 8/0 · 회복 10/0 · 점술 장식). 빌드 다양성은 카드 수가 아니라 **참조 밀도**에서 나온다(Balatro 조커의 40~45%가 명시 참조, StS 프로스트 오브의 교훈 "payoff 빈곤이 축을 죽인다"). 처방은 3층위(① 카드 payoff ② 귀신 팩션 문법 ③ 보상 경제) — 설계 단계에서 적대 밸런스 검토(레이어 결합 폭주 8건 발견·처방)와 구현 비용 검증을 선행했고, **레이어를 같은 릴리스에 싣지 않는다**(각자는 안전한 안이 합쳐질 때 터지는 유형 — 교차 반입마다 재계측). v0.8은 ①의 P0분.

- **무소속 무구 강화 개통 (버그 수정)**: `isUp()`이 owner 없는 카드를 무조건 걸러 무구 32장의 `u:` 강화 데이터가 전부 사장 상태였다 — 인스턴스 `up` 플래그 도입(저장 직렬화 자동 호환, mapVersion 불변). 획득 접점(장터 덧칠·치성 확장)은 경제 레이어(P1)에서. runStats `upgradedCards`도 isUp 단일 기준으로 통일

- **자해 축 → 제물 자원화**: `onSacrifice()` 단일 창구(전투 누적 `battleSelfAcc`). **제물 = 실제 잃은 체력만** — 방어에 흡수된 자해는 세지 않는다. 작두 타기 리워크(2코 18뎀 바닐라 → 12뎀 + 제물 1당 +2, 상한 +12 — 기본 −6/스케일 +12 등가 재배분) / 핏값(1코 소멸 신령부: 제물 바친 턴 힘 +1, **턴당 1회** — 사기(士氣) 전례)

## v0.3 (무속 설정 심화 + 주체적 서사 + 전략 확장)

- **무속 용어 체계**: 런 = 밤길의 **진오기굿** / 막 = **부정거리 → 영실거리 → 뒷전거리** / 에너지 = **신력(神力)** / 상점 = 당골네 할멈의 장터(엽전) / 반복 난이도 = **금제(禁制)** 0~3
- **업(業) 시스템**: 봉인 = 신위(神威)+2, 천도·위령제 = 공덕(功德)+. 업에 따라 **무복 색이 변하고**(밤빛↔새벽빛) **엔딩이 3갈래**로 갈린다: 흑무(귀문의 새 주인) / 성무(큰무당) / 만신. 원한 12 초과 시 **역빙의** — 가장 사나운 봉인 귀신이 몸을 노리는 강제전 (패배 시 전용 배드엔딩)
- **해원(解冤) 퀘스트**: 신물(족두리·꽃신·제삿밥)을 구해 사연이 맞는 귀신에게 건네면 전투 없이 **해원 봉인(원한 0)** 또는 **해원 천도(대회복)**. 혼수함 이벤트·제삿밥 짓기로 신물 획득, 지도에 '解' 표시
- **굿거리(궁극기)**: 카드 5장 = 신명(神明) 1게이지 → 빙의 중인 신령의 대목 발동. 17종 전부 고유 (참수/초혼: 죽음 무효/씨름 한판: 적 의도 넘어뜨림/명부 집행: 비례 피해/독 옮기기: 恨 전이…)
- **경로 예고**: 지도에 다음 귀신의 이름·변종이 미리 보인다 — 신물 들고 해원 상대를 찾아가는 루트 플레이
- **귀신 변종**: 굶주린(흡혈)/성난(공격+2)/묵은(체력 1.35배)
- **장터**: 엽전(전투 보상)으로 부적·무구·명다리·수리취떡·소각 구매
