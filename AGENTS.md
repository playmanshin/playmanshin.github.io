# manshin — AI 에이전트 작업 가이드 (Codex 등)

한국 무속 호러 덱빌딩 로그라이크. 단일 파일 웹 게임.

## 실행·검증

```bash
open index.html            # 실행 (서버 불필요, 모바일 세로 360×640 기준)
cd tests && ./run.sh       # 회귀 스위트 — ALL_GREEN 필수
```

- 테스트는 index.html에서 `<script>`를 추출(`tests/_game.js`, gitignore됨)해 DOM/Canvas 스텁 위에서 실행한다. 각 스모크의 대상은 `tests/README.md` 참조.
- 모든 검증은 `node:assert/strict` — 값이 틀리면 스위트가 실패한다. 새 기능/수정에는 대응 assertion을 추가할 것.

## 리뷰·수정 시 주의점

- **피해 계산은 `calcIncomingHit()` 한 곳** — 전투 실행과 의도 표시(칩·상세)가 공유한다. 피해 관련 수정은 반드시 이 함수를 거치고, 다단히트는 행동 개시 시 1회 스냅샷 규칙을 유지할 것.
- **밸런스 상한선들은 의도된 설계다** (적대 밸런스 검토로 확정): 恨 상한 18/24, 굿거리 턴당 1회, 낙인 카드당 1회·상한 8, X 최대 4타, 방어이월 상한 12, 신령부 중첩 금지. 완화하려면 GDD의 해당 검토 항목을 먼저 반박할 것.
- **출전 체계**: `party[].active`가 전투 덱(`battleDeck()`)·강신·굿거리를 결정한다. 대기 신령도 원한은 유지된다(의도).
- **저장 호환**: 런 저장 스키마를 바꾸면 `mapVersion`을 올려라. 구버전은 로드 시 폐기된다.
- **지도 그래프 불변식**: 모든 노드는 시작점에서 도달 가능, 막다른 길 없음 (smoke11이 200회 생성 검증). `genMap` 수정 시 이 테스트를 반드시 돌릴 것.
- **아트 규칙**: 캐릭터는 fillRect 블록 매스만(arc/ellipse 금지). 색은 무채색 위주 + 강조 원색 1-2곳 — 렌더 파이프라인이 무채색을 4단 흑백램프+디더링으로 바꾼다.
- 게임 내 텍스트는 한국어, 무속 용어 체계는 CLAUDE.md의 용어표를 따른다.
